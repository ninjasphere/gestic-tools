gestic_t Struct

The single most simple use of the SDK is the following (it works only with the Dynamic API as gestic_version_str is not available in the Static API, the GESTIC_VERSION_STRING macro could be used there instead):

#include <gestic_api.h>
#include <stdio.h>
int main()
{
    printf("Using the GestIC SDK %s\n", gestic_version_str());
    return 0;
}

Applications always include gestic_api.h. gestic_api.h in turn includes the other headers depending on the settings.  Including one of those headers directly will likely result in compiler errors.

Example

#include <gestic_api.h>
#include <stdio.h>

int main()
{
    // Create gestic_t instance
    gestic_t *gestic = gestic_create();
    if(!gestic) {
        fprintf(stderr, "Could not allocate gestic_t instance\n");
        return -1;
    }

    // Initialize gestic
    gestic_initialize(gestic);

    // Connect to device
    if(gestic_open(gestic) != GESTIC_NO_ERROR) {
        fprintf(stderr, "Could not connect to GestIC device\n");
        return -1;
    }

    printf("Succesfully connected to GestIC device\n");

    // Use gestic...

    // Disconnect from device
    gestic_close(gestic);

    // Cleanup
    gestic_cleanup(gestic);

    // And finally freeing the gestic_t instance
    gestic_free(gestic);

    return 0;
}

gestic_t instance

The GestIC API is reentrant with almost all functions requiring a pointer to an gestic_t-instance.  This instance can be created with gestic_create and has to be freed later with gestic_free.

Instances have to be initizalized before usage with gestic_initialize.  Once they are no longer needed they have to be cleaned up before freeing with gestic_cleanup.

gestic_t *gestic = gestic_create();
if(gestic == NULL) {
    // TODO Handle error
}
gestic_initialize(gestic);

// Use gestic...

gestic_cleanup(gestic);
gestic_free(gestic);

Connecting to device

Connecting to the device gets done by calling gestic_open.  This connection is later released with gestic_close.

Depending on the actual IO-implementation (See Custom IO Implementation) there might be additional functions that could be called to configure the connection before gestic_open.
