Data Retrieval and Real Time Control

The primary taks of the MGC3130 is detection of Position and/or Gestures.  The result of those detections could be fetched with the data retrieval feature of the SDK.

Data Retrieving

New data could be retrieved with a call to gestic_data_stream_update which updates the internal result buffers.

The buffers for the individual data could be get with the gestic_get_* functions.  The content of those buffers are updated when calling gestic_data_stream_update when there were new values or events.  What kind of data gets communicated could be controlled with gestic_set_output_enable_mask.

Note

The Static API provides direct access to the result buffer and doesn’t provide the gestic_get_* functions.

Here is an overview of the possible data buffers and how to acces them:

The Stream Examples (Windows/Linux) or Stream Example (PIC32) contain complete examples of how to retrieve data with different configurations.

Real Time Control

Real Time Control functions allow to change GestIC parameters during runtime and trigger special acctions.

Parameters could be read with gestic_get_param and gestic_set_param.  gestic_parameter_id_t is an enumeration of the available parameters.  A complete description of them could be found in the MGC3130 GestIC Library Interface Description User’s Guide.

While all parameters could be controlled with gestic_get_param and gestic_set_param some parameters have special helper functions with an easier to use signature.  Those parameters are:
