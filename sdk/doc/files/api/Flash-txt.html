Introduction

After Power-on or hardware Reset the MGC3130 executes the Library Loader routine.  First it sends a Fw_Version_Info message to the host and starts a timer.  After a time-out of 100 ms, the GestIC Library is started automatically.

Before the timer timed-out the GestIC Library update can be initiated via the Fw_Update_Start message.  Once started the Library Loader stays in update mode until the next Reset of the MGC3130.

The update itself is started with the Fw_Update_Start message followed by a sequence of FW_Update_Block message and finished with a Fw_Update_Complete message.

An update of the Library Loader itself gets done by replacing the GestIC Library with a Library Loader Updater.  After the next reset this Updater is started instead of the GestIC Library and updates the Library Loader.  During this procedure the Library Loader Updater sends multiple Fw_Version_Info messages with feedback about the progress.

Afterwards the Updater has to be replaced with the GestIC Library again to have an fully operational system.  The following section describes how this process could be done with the help of the GestIC SDK.

Acquiring Firmware Version

The currently running firmware could get fetched with gestic_query_fw_version.

char fw_version[120];
int error;

error = gestic_query_fw_version(gestic, fw_version, sizeof(fw_version), 100);
if(error != GESTIC_NO_ERROR) {
    fprintf(stderr, "Error: Could not read firmware version\n");
    return error;
}

printf("Currently running GestIC Library: %s\n", fw_version);

Using gestic_flash_image

The easiest way to do an update is calling gestic_flash_image.  This function does the whole procedure of updating the firmware from a first Reset to sending the final Fw_Update_Complete message with the instruction to do one more Reset.

gestic_flash_image expects an instance of gestic_flash_image_t as an argument which contains the image to be flashed.  This image can be extracted from ENZ-files with the help of MGCConv.

MGCConv is a tool which can create self-contained source-files with the definition of a global variable containing the image.  The structure used in this declarations is compatible with gestic_flash_image_t.  Therfore this variables could easily be used by forward declaring those variables as extern or by casting them to gestic_flash_image_t.  This allows to simply add those source-files to a project and use the contained images to update the firmware.

// Update of the GestIC Library alone.

extern gestic_flash_image_t Library;

int flash_image(gestic_t *gestic)
{
    int session_id = 1; // Some random value
    int error;
    error = gestic_flash_image(&gestic, session_id, &Library,
                               gestic_UpdateFunction_ProgramFlash, 100);
    if(error != GESTIC_NO_ERROR)
    {
        fprintf(stderr, "Error: Could not flash image\n");
        return error;
    }
    return GESTIC_NO_ERROR;
}

Updating the Library Loader requires one call to gestic_flash_image to write the Library Loader Updater.  Before continuing with the update of the GestIC Library the application has to wait until the Updater has updated the Library Loader which requires about 20 seconds.

The function gestic_flash_wait_loader_updated blocks until the Library Loader Updater has finished the update and sent the appropriate Fw_Version_Info message.

Afterwards a second calls to gestic_flash_image is needed to replace the Updater with the GestIC Library again.

// Complete update including the Library Loader and the GestIC Library

extern gestic_flash_image_t Loader;
extern gestic_flash_image_t Library;

int flash_image(gestic_t *gestic)
{
    int session_id = 1; // Some random value
    int error;
    error = gestic_flash_image(&gestic, session_id, &Loader,
                               gestic_UpdateFunction_ProgramFlash, 100);
    if(error != GESTIC_NO_ERROR)
    {
        fprintf(stderr, "Error: Could not flash Library Loader Updater\n");
        return error;
    }

    error = gestic_flash_wait_loader_updated(&gestic, 20000);
    if(error != GESTIC_NO_ERROR)
    {
        fprintf(stderr, "Error: Library Loader Updater failed to update Loader\n");
        return error;
    }

    session_id = 2;
    error = gestic_flash_image(&gestic, session_id, &Library,
                               gestic_UpdateFunction_ProgramFlash, 100);
    if(error != GESTIC_NO_ERROR)
    {
        fprintf(stderr, "Error: Could not flash GestIC Library\n");
        return error;
    }
    return GESTIC_NO_ERROR;
}

Sending messages directly

The disadvantage of gestic_flash_image is that it blocks until the flash process was finished without any progress indicator.  An alternative is to use the underlying functions directly.

gestic_flash_begin starts the flash process by resetting the device and sending the Fw_Update_Start message.  The individual blocks could be sent by a call to gestic_flash_write.  gestic_flash_end finalizes the flash process by sending Fw_Update_Completed twice: Once with the instruction to store the new Firmware version and once with the instruction to do a reset.

Note

The reset triggered by gestic_flash_image or gestic_flash_end is executed after a delay of several seconds.

The implementation of gestic_flash_image or the source-code of the Programmer Application (Windows/Linux) or Programmer Application (PIC32) for embedded platforms could be used as a reference of how to use the low-level gestic_flash_* functions directly.

int gestic_flash_image(gestic_t *gestic,
                       unsigned int session_id,
                       gestic_flash_image_t *image,
                       gestic_UpdateFunction_t mode,
                       int timeout)
{
    int error = GESTIC_NO_ERROR;
    int i;
    gestic_flash_record_t *record;

    error = gestic_flash_begin(gestic, session_id, image->iv, mode, timeout);

    for(i = 0; !error && i < image->record_count; ++i) {
        record = image->data + i;
        error = gestic_flash_write(gestic, record->address, record->length,
                                   record->data, mode, timeout);
    }

    if(!error)
        error = gestic_flash_end(gestic, image->fw_version, timeout);

    return error;
}

Verification

The gestic_flash_* functions have a second mode of operation: gestic_UpdateFunction_VerifyOnly.  In this mode the records are send for verification of the existing firmware only.

If a session was started by providing gestic_UpdateFunction_VerifyOnly to gestic_flash_begin no call to gestic_flash_write with a different update function is allowed.

However it is possible to call gestic_flash_write with gestic_UpdateFunction_VerifyOnly during a session started with gestic_UpdateFunction_ProgramFlash.  This could be used to verify the data on the chip immediately after writing them.
