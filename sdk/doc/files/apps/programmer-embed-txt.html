Programmer Application (PIC32)

The programmer application is a self-contained firmware updater for the PIC32 USB Starter Kit II (http://www.microchip.com/DevelopmentTools/dm320003-2) and the I/O Expansion Board (http://www.microchip.com/DevelopmentTools/dm320002).

It’s source-code together with a MPLAB X project is located at apps\PIC32\programmer.

The files Library.c and Loader.c contain the firmware images in form of a gestic_flash_image_t-compatible variable declaration.  To create this representation for different firmware files the GestIC SDK provides the MGCConv-tool.

Usage

This application uses the following connection between the PIC32 and the MGC3130

MGC3130PIC32Expansion Board
TS-LineRE8J10.37
Reset-LineRE6J10.6
I2CI2C3
-> SDASDA3/U1RXJ11.41
-> SCLSCL3/U1TXJ11.43

This configuration is the default one used in the example.  It could be changed by changing the DEV0_* defines in HardwareProfile.h or by providing an alternative to gestic_dev0_handler in gestic_custom.c.

Once started the buttons 1-3 can be used to trigger the differen actions

Button 1 (SW1) executes an hardware reset of the MGC3130.

Button 2 (SW2) starts the update of the Library Loader followed by an update of the GestIC Library.

Button 3 (SW3) starts the verification of the GestIC Library.

Please see Firmware Version and Update for a introduction to firmware update.

Implementation

This application provides a custom IO-implementation in gestic_custom.h and gestic_custom.c.  This implementation communicates with the MGC3130 via the I2C3 ports.  This example uses polling to receive messages for simplicity.
